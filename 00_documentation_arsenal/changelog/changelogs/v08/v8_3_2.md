## 📌 1) Template sensors — Factorisation interne (ancres YAML)

- Refonte ciblée de capteurs répétitifs via **logicique factorisée par ancres YAML** (moteurs uniques réutilisés par alias).
- Déduction systématique des entités sources via `this.entity_id` (approche par suffixes).
- Réduction de duplication → baisse du risque d’incohérence inter-capteurs, maintenance simplifiée, lisibilité accrue.
- **Invariant déclaré :** maintien strict du périmètre fonctionnel existant (entités, sorties, logique) — réorganisation exclusive du code.

### Applications notables (signal net)
- ΔT : remplacement d’expressions répétées par une logique unique factorisée.
- Météo / points de rosée : passage d’une logique copiée/collée vers un moteur commun.
- Couleurs / humidité absolue : factorisation du calcul et généralisation par suffixes.
- Humidex (seuil haut) : factorisation du calcul (moteur unique).
- Programme chauffage : **icône dynamique** selon le programme ViCare actif (Confort / Eco / Inconnu) pour alignement UI ↔ état réel.

## 📌 2) Gouvernance de contexte / ECS — Cycle Vacances / Normal / Désinfection

Axe fort : recentrage des automations “modes maison” sur leur rôle légitime, avec périmètre central **contexte ECS**.

### Entrée Vacances — Automation `1009000000006`
- Suppressions :

  - Climatisation forcée
  - Sécurisation ECS à 10 °C
  - Reset timers ECS

  - `ecs_blocage_planifiee` → ON

  - `mode_vaisselle` → OFF
  - `bouclage_visiteur` → OFF

  - Vacances = gel ECS + suspension usages + marqueur UI

### Sortie Normal — Automation `1009000000007`

  - `ecs_blocage_planifiee` → OFF

  - reset présence
  - forçage modes

## ⚠️ 5) Risques & points d’attention (technique)

- **Ancres YAML** : un moteur commun cassé = impacts transverses sur plusieurs capteurs.
- **Déduction par suffixes** : suffix inattendu / renommage futur → entité source mal résolue.
- **Déplacements d’arborescence** : risque de références résiduelles à l’ancien chemin (includes, docs, grep).
- **Modes maison simplifiés** : révélation possible de dépendances implicites (des éléments comptaient sur des actions annexes désormais supprimées).
