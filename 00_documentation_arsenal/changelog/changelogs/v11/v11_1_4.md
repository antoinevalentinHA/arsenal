## Signal net

- **Chauffage + ECS : déploiement du retry transactionnel** — infrastructure complète ex nihilo : `input_number.retry_count`, `input_text.retry_attempt[1,2]_id`, `input_boolean.retry_pending`, `timer.retry`, `input_select.chauffage.mode_session` ; pipeline de 4 automations par domaine (`armement` → `déclenchement` → `état` → `reprise`) ; pattern symétrique chauffage/ECS. Le retry est une **couche externe de résilience** : aucune logique de retry dans les scripts exécutifs, aucune altération du modèle ACK, orchestration entièrement observable via helpers.

  > Le retry transactionnel ne constitue pas un chemin nominal.  
  > En régime nominal, toute exécution aboutit sans retry.  
  > Le retry intervient uniquement en cas d'échec explicite ou d'absence d'ACK.

- **Alarme : refonte du watchdog sur déclencheur structurel** — le watchdog `10020000000034` ne surveille plus directement `input_boolean.blocage_armement_auto` + `timer.blocage_armement_auto` ; il se déclenche désormais sur `binary_sensor.blocage_armement_incoherent` (`off → on`, `for: 500 ms`) ; la stabilisation est portée par le déclencheur, plus par un `delay` interne. Alignement strict du principe Arsenal : **diagnostic → action**, sans logique embarquée.

- **Alarme : diagnostic structurel du couple blocage ↔ timer** — ajout de `binary_sensor.blocage_armement_incoherent` (détection des deux états impossibles : blocage ON + timer idle ; blocage OFF + timer actif/paused) ; contrat `96_diagnostic_blocage_armement_incoherence.md` ; invariant 2 du contrat `60_delais_et_blocages.md` étendu de `active` à `{active, paused}`.

- **Recorder : traçabilité de l'auto-ajustement chauffage** — ajout de `sensor.ecart_consigne_instantane`, `sensor.ecart_consigne_instantane_froid`, `sensor.ecart_consigne_instantane_doux` en Population A ; historique des écarts consigne disponible pour l'analyse des cycles d'auto-ajustement.

- **Boiler : normalisation documentaire des contrats** — renommage kebab-case : `CONTRAT_BOILER_SOCLE_TRANSACTIONNEL.md` → `socle_transactionnel.md`, `CONTRAT_MQTT_ACK_HA.md` → `mqtt_ack_ha.md`, `CONTRAT_SCRIPT_EXECUTIF.md` → `script_executif.md` ; suppression des anciens fichiers en CASSE. Aucun changement fonctionnel.

- **Alarme : normalisation des noms de contrats** — `51_ouvrants_entree_alarme.md` → `51_ouvrants_entree.md`, `90_ui_alarme.md` → `90_ui.md` ; suppression du suffixe redondant avec le dossier parent.

## Invariants maintenus

- Aucun succès sans ACK `applied`
- Aucun retry implicite dans les scripts exécutifs
- Aucune logique décisionnelle dans les scripts exécutifs
- Séparation stricte : décision / exécution / diagnostic / résilience

---

## 🚫 Refus explicites

Cette version introduit des garanties fortes sur ce que le système ne fait pas :

- Aucun retry implicite dans les scripts exécutifs
- Aucune boucle interne aux scripts
- Aucune inférence de succès par télémétrie
- Aucun contournement du modèle ACK
- Aucun couplage entre retry et décision centrale

Le retry est strictement : externe, borné, observable, découplé de l'exécution.

Toute dérive vers un retry implicite invalide le modèle transactionnel.
