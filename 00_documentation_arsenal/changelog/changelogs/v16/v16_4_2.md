# Climatisation

## Abstention native de l'état rapporté (C30)

- `sensor.clim_mode_local` : ajout d'un bloc `availability` (liste blanche `off` / `cool` / `dry` / `heat` / `auto` / `fan_only` sur `climate.clim`) ; suppression du double repli `this.state` (état et icône) et du fallback terminal `off`.
- `sensor.clim_action_en_cours` : ajout d'une garde `availability` (`unknown` / `unavailable`) ; redéfini comme « état rapporté par l'intégration ».
- `binary_sensor.clim_incoherence_decision_reel` : ajout d'une garde `availability` exigeant les trois opérandes exploitables (`sensor.clim_target_mode`, `switch.clim_power`, `sensor.clim_mode_local`) ; abstention si l'un manque. `for: 00:01:00` inchangé.
- `sensor.etat_clim_dashboard` : ajout d'une garde `availability` (liste blanche sur `sensor.clim_mode_local`) ; ne fabrique plus `off`.
- Cartes `carte_clim_synthese` et `carte_clim_decision` : garde d'indisponibilité (`unknown` / `unavailable` / `none` / vide) → `Indisponible`, icône `mdi:help-circle-outline`, fond `rgba(158, 158, 158, 0.1)` prioritaire ; plus de flocon COOL par défaut.

## Notification d'échec d'exécution (C30)

- `notification_echec_execution.yaml` : message reformulé neutre, ne qualifie plus l'état de l'infrastructure Airstage. Triggers, conditions et `notification_id` (`clim_execution_echec`) inchangés.

## Seuils COOL (C20)

- `sensor.seuil_allumage_clim_applique` et `sensor.seuil_extinction_clim_applique` : ajout d'un trigger `event_template_reloaded`.
- Dépendance `binary_sensor.presence_famille_unifiee` remplacée par `binary_sensor.presence_confort_thermique_stabilisee` ; ajout de `binary_sensor.clim_mode_nuit_effectif`.
- Seuil « présence » retenu si présence thermique stabilisée active et mode nuit inactif ; seuil « absence » sinon.

## Contrats

- `10_decision.md`, `20_chaines.md`, `10_observabilite.md` : `clim_mode_local` et `clim_action_en_cours` redécrits en abstention native, ordre « mode HVAC actif rapporté > blocage poêle > arrêt ».
- `capteurs/coherence/00_overview.md` et `10_coherence.md` : ajout du « CAS 0 » — un opérande non exploitable → abstention native, aucun verdict rendu.
- `08_execution.md` : cause reconstruite sans état durable, clause de message neutre ; politique de reprise (+30 s puis +90 s, 2 reprises) inchangée.
- `seuils_et_franchissements/10_sensors_seuils.md` : nouvelles dépendances et triggers des seuils COOL.

## Contrôles

- `check_climatisation_admissibilite_contracts.py` : ajout de 4 tests (abstention native de `sensor.clim_mode_local`, `binary_sensor.clim_incoherence_decision_reel`, `sensor.etat_clim_dashboard` et `carte_clim_synthese` ; message neutre de la notification d'échec) ; test de `sensor.clim_action_en_cours` durci (présence du bloc `availability`, ordre « mode HVAC actif > blocage poêle »).
