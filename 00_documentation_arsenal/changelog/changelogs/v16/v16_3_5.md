# Arrosage — supervision des exécutions longues Rain Bird

La station 1 était pilotée par un `delay` long dans le script. Le cycle est remplacé par une session persistée : start court, fin portée par une automation, stop supervisé sur preuve du switch natif.

## Session persistée (helpers ajoutés)

- `input_number.arrosage_session_duree_minutes` (`03_input_numbers/arrosage/session.yaml`) — durée de la session en cours, écrite par le script à l'ouverture, sans `initial` (distinct du réglage `arrosage_rainbird_station_1_duree_minutes`).
- `input_text.arrosage_session_verdict` (`04_input_texts/arrosage/session.yaml`) — discriminant de session : `ouverte`, `demarrage_non_confirme`, `fin_observee`, `close_nominale`, `close_reprise`, `close_watchdog`, `stop_incertain`.
- `input_datetime.arrosage_session_debut`, `..._fin_prevue`, `..._fin_observee` (`07_input_datetimes/arrosage/session.yaml`).

## Lectures dérivées (ajout)

- `12_template_sensors/arrosage/session.yaml` : `binary_sensor.arrosage_session_en_cours` (`on` si verdict `ouverte`) et `sensor.arrosage_session_etat` (`aucune` / `ouverte` / `depassement` / `fin_observee` / `close_nominale` / `close_reprise` / `close_watchdog` / `stop_incertain` / `close_anomalie`, marge de dépassement 300 s).

## Automations de fin (ajouts)

- `11_automations/arrosage/session_fin_observee.yaml` (id `10270000000005`, `mode: single`) — sur transition `switch.rain_bird_bat_bt_2_e9a3_station_1` de `on` à `off` et verdict `ouverte` : horodate `..._fin_observee` et pose le verdict `fin_observee`.
- `11_automations/arrosage/session_fin_watchdog.yaml` (id `10270000000006`, `mode: single`, `max_exceeded: silent`) — triggers sur l'échéance `..._fin_prevue`, `homeassistant.start` et `time_pattern` /5 min : appelle `script.arrosage_rain_bird_stop_supervise` si le switch n'est pas `off`, clôture avec le verdict cible selon le trigger, notifie seulement hors chemin nominal. Seul porteur de la fin de session.

## Scripts (modifications)

- `10_scripts/arrosage/station_1_courte_supervisee.yaml` : suppression du `delay` long et du stop de fin → start court (< 30 s). Garde de repos durcie (exige `switch.rain_bird_bat_bt_2_e9a3_station_1 == 'off'` en preuve primaire, en plus des préconditions). Ouvre la session avant la commande native ; verdict `demarrage_non_confirme` si le démarrage n'est pas confirmé.
- `10_scripts/arrosage/stop_supervise.yaml` : preuve d'arrêt basée sur `switch.rain_bird_bat_bt_2_e9a3_station_1 == 'off'` au lieu de `active_station` ; `unavailable` / `unknown` ne valent pas `off` ; boucle bornée à 3 essais.
- `11_automations/arrosage/declenchement.yaml` (id `10270000000002` inchangé) : ajout de `max_exceeded: silent` (recouvrement des déclencheurs à l'entrée en fenêtre silencié).

## Divers arrosage

- `12_template_sensors/arrosage/demande_climatique.yaml` : sources VPD/ET₀ recâblées de `sensor.temperature_exterieur` / `humidite_relative_exterieur` vers `sensor.temperature_jardin` / `sensor.humidite_relative_jardin`.
- `01_customize/connectivite/bluetooth_signal.yaml` : customize de `sensor.rain_bird_bat_bt_2_e9a3_ble_rssi` (nom, icône).
