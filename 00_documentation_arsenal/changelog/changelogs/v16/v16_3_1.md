# Arrosage

## Décision — mode dégradé exploitable (Option A)

- `12_template_sensors/arrosage/reservoir_sol.yaml` : disponibilité de `sensor.jardin_humidite_sol_mediane` abaissée de `>= 3` à `>= 2` points frais. À 2 points, médiane = moyenne des deux valeurs.
- canal `degrade` (2/3 points frais) désormais recevable pour `binary_sensor.arrosage_intention` ; abstention conservée à 1/3 et 0/3.
- `15_canal_reservoir_sol.md` (§2/§5) et `17_decision_v1.md` (§3.7) mis à jour : 2/3 = dégradé exploitable, médiane disponible dès 2 points.
- `17_decision_v1.md` : ajout de l'encart « Réalisation runtime V1 » (table rôles → entités réelles) ; statut passé de « runtime non livré » à « runtime V1 mergé, publié v16.3 ».
- `contrats/arrosage/README.md` : statut du `17` passé à « runtime livré (PR #125, publié v16.3) ».
