Correction d'un **décalage de contenu** dans la chaîne contractuelle Alarme : chaque fichier de `20` à `40` portait le contenu du slot précédent (en-tête et `Chemin` internes déjà labellisés pour ce slot précédent), et la couche **Application de la décision** (slot `40`) était **absente de l'arbre**.

Origine établie par l'historique git : le commit `4bbde856` (« contrats alarme 20/30/40 et évolutions futures ») avait **correctement** réorganisé 20/30/40 **et** créé la couche Application ; il a été **annulé par un revert trop large** (`c42033e8`, ~5 min après) qui visait `evolutions_futures/github.md` mais a emporté collatéralement la réorganisation 20/30/40 et le contenu Application. Les commits ultérieurs (`normalize H1`, `line endings`) n'ont fait que **préserver** ce décalage.

Remise en cohérence (rotation de contenu, **noms de fichiers préservés**) :

- `contrats/alarme/20_interfaces_contexte_et_helpers.md` : reçoit le contenu **Interfaces, contexte & helpers** (auparavant logé dans le fichier `30`). Remplace un **duplicata du contrat `10`** (Modèle d'états) qui occupait ce fichier à tort.
- `contrats/alarme/30_decision_centrale.md` : reçoit le contenu **Décision centrale (pure)** (auparavant logé dans le fichier `40`).
- `contrats/alarme/40_application_decision.md` : reçoit le contenu **Application de la décision** restauré depuis `4bbde856` (couche d'application unique, `automation.alarme_decision_centrale`, ID `10020000000027` — confirmé au runtime), normalisé (LF, H1 single-line, `Chemin` au format lien).

Aucune vérité artificielle créée : le contenu Application provient d'une version **authentique** de l'historique git, fidèle au runtime actuel.
