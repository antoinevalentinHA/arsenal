## 1 — Chauffage : filtrage des épisodes d'aération non qualifiés

Correction principale de la v9.4.1.

L'automation `11_automations/chauffage/decision_centrale_trigger.yaml` est corrigée pour ignorer les transitions d'aération non confirmées. 
Un filtre `choose` est introduit en tête d'action pour intercepter les cas :

- ouverture tentative (`aeration_episode_en_cours` off → on, `aeration_confirmee` off)
- clôture tentative (`aeration_episode_en_cours` on → off, `aeration_confirmee` off)

Le trigger `aeration_episode_en_cours` reçoit l'identifiant `aeration_episode` pour permettre le routage conditionnel. 
Le trigger strict `from: off` est également introduit pour sécuriser la détection d'entrée d'épisode.

## 2 — Aération : script M0 — autorité de remédiation externe

Introduction du script de remédiation normative :

```
10_scripts/aeration/m0_remediation_incoherence.yaml
```

M0 n'est pas un maillon du pipeline M1–M5 : c'est une **autorité de remédiation externe**, invoquée en cas d'état incohérent au démarrage. 
Il couvre les cas pathologiques : pipeline zombie, confirmation orpheline, blocage orphelin.

Le template sensor `coherence.yaml` reçoit un invariant supplémentaire pour renforcer la détection des états KO.
