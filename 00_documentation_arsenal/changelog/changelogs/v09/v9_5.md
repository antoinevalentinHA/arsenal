## 1 — Aération : paramétrisation complète de l'analyse ΔT (M3)

Le script M3 d'analyse thermique n'utilise plus de valeurs hardcodées. 
Les seuils et durées de prolongation du blocage chauffage sont désormais configurables via helpers.

### Nouveaux paramètres

```
03_input_numbers/aeration/blocage_chauffage/parametrage_prolongation/seuils/tiny.yaml
03_input_numbers/aeration/blocage_chauffage/parametrage_prolongation/seuils/medium.yaml
03_input_numbers/aeration/blocage_chauffage/parametrage_prolongation/seuils/high.yaml
03_input_numbers/aeration/blocage_chauffage/parametrage_prolongation/durees/tiny.yaml
03_input_numbers/aeration/blocage_chauffage/parametrage_prolongation/durees/medium.yaml
03_input_numbers/aeration/blocage_chauffage/parametrage_prolongation/durees/high.yaml
```

Chaque niveau (tiny / medium / high) définit un seuil ΔT et la durée de prolongation associée. 
La durée de prolongation passe d'une logique en heures entières à une logique en minutes flottantes (`float(0)` au lieu de `int(0)`).

### Traçabilité décisionnelle

Deux helpers capturent les valeurs effectivement utilisées lors du blocage :

```
03_input_numbers/aeration/blocage_chauffage/delta_t_utilise.yaml
03_input_numbers/aeration/blocage_chauffage/reference_thermique_utilisee.yaml
```

## 2 — Aération : guard dur M2 sur ouvertures actives

Le script `10_scripts/aeration/m2_fin_episode.yaml` introduit un **guard dur** en tête d'exécution : 
M2 est refusé si une ouverture est encore détectée active au moment de la clôture d'épisode.

Conditions vérifiées :

```
binary_sensor.fenetre_ouverte_maison
binary_sensor.fenetre_ouverte_maison_avec_delai
```

Si une ouverture est encore active : M2 s'arrête, aucun timer de blocage n'est lancé, un log explicite est enregistré. 
Objectif : éliminer les états « M2 fantôme ».

## 4 — VMC : clarification de responsabilité des scripts

Les scripts `vmc/haute_vitesse.yaml` et `vmc/basse_vitesse.yaml` suppriment toute référence à `input_boolean.vmc_haute_vitesse`. 
Les scripts exécutent l'action physique uniquement — la vérité système appartient aux automatisations. 
La nouvelle automation `11_automations/vmc/synchro_booleen.yaml` prend en charge la synchronisation du booléen.

## Synthèse

La v9.5 complète le pipeline aération sur deux axes : paramétrisation complète de l'analyse ΔT en M3, 
et guard dur en M2 contre les clôtures d'épisode sur ouvertures actives.

La réorganisation de l'arborescence changelog finalise la structure documentaire versionnée.
