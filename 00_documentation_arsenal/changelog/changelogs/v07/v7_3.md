### 🔔 ECS — Signal canonique de fin de cycle (production / consommation)

Problème traité : l'auto-ajustement des offsets ECS était déclenché par des écritures de helpers "figés" — 
signal implicite, risque de rejeu, ambiguïté sur la notion de "fin de cycle".

**Signal introduit :** `input_boolean.ecs_fin_cycle_signal` — événement explicite, unique, consommable.

Séparation stricte production / consommation :

- **Production** : nouvelle automation `signal_fin_cycle.yaml` (ID `10130000000099`) — 
déclenchée sur écriture d'une nouvelle durée figée valide. Rôle exclusif : émettre le signal.
- **Consommation** : `auto_ajustement_offset.yaml` (ID `10130000000088`) — 
déclenchement exclusif sur `ecs_fin_cycle_signal → on`. 
Séquence : appel `pyscript.ecs_autocorrect_offsets` → **ACK** (`signal → off`). Si échec → pas d'ACK → signal reste ON (rejouable, investigation possible).

Fin de cycle ECS : explicite, robuste au redémarrage, contractualisé "événement + ACK".

### 🔥 Chauffage — Verrou décisionnel en présence

Problème traité : en présence, le système pouvait rester en Neutre/Eco malgré un besoin thermique réel — 
court-circuit logique inhibant une chauffe légitime.

**Principe renforcé :** la présence n'est pas une autorité thermique — elle accorde uniquement le **droit de décider**. 
L'autorité thermique est `sensor.chauffage_autorisation_cible`.

Correctif dans `decision_centrale.yaml` :
- En présence, le mode désiré est désormais dérivé de `sensor.chauffage_autorisation_cible` (et non d'un raccourci `presence_on`).
- La raison associée est lue depuis `sensor.chauffage_raison_calculee`.
- Suppression de la branche `binary_sensor.pre_confort_actif`.
- Simplification du cas absence (retour `absence`).

### 🧠 Chauffage — Alignement capteurs diagnostic ↔ décision ↔ UI

`sensor.chauffage_mode_calcule` repositionné explicitement comme capteur **diagnostique** — lisible, 
non consommable comme autorité d'action.

Hiérarchie explicitée dans `mode.yaml` et `raison.yaml` :
1. Override utilisateur (`input_boolean.mode_confort_chauffage`)
2. Blocages majeurs (système, aération, fenêtres, vacances, poêle)
3. Présence → mapping depuis `sensor.chauffage_autorisation_cible` (Confort / Neutre / Eco)
4. Absence → Eco

Suppression de toute divergence décision ↔ capteurs ↔ UI.
