### 🧠 Ouvertures — Moteur de réconciliation : extraction vers scripts spécialisés

Le moteur `11_automations/ouvertures/moteur_capteurs_redondants.yaml` est transformé en **orchestrateur pur** : 
toute la logique métier inline (~260 lignes de `choose` imbriqués) est extraite dans cinq scripts dédiés.

**Scripts introduits (`10_scripts/ouvertures/reconciliation_redondance/`) :**
- `traiter_source` — transitions `on`/`off` des sources brutes `_1`/`_2`
- `traiter_expiration` — expiration timer de corroboration
- `inhiber_tous` — passage global en inhibition sur `systeme_stable = off`
- `lever_inhibition` — levée d'inhibition sur `systeme_stable = on`
- `appliquer_transition` — application de la transition d'état réconcilié

L'automation ne contient plus aucune logique métier : elle route chaque trigger (sources brutes, timers, signal stabilité) 
vers le script correspondant. Mode `queued` conservé pour sérialiser les événements concurrents.

Périmètre inchangé : `entree_porte`, `chambre_parents_*` (×3), `garage`, `sejour_3`, `chambre_arnaud`, `chambre_matthieu`.

Le contrat v2.2 n'est pas modifié. Seul le *où la logique vit* change.
