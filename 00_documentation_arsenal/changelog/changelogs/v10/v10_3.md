# Changelog Arsenal v10.3

Statut : PUBLIÉ
Nature : **Simplification infrastructure / Robustesse ECS / Préconfort vacances / Autorisation éclairage garage**
Portée : ECS, modes vacances, infrastructure scripts, éclairage garage, alertes Vicare

## 1 — Suppression complète de Pyscript

La v10.3 supprime entièrement la dépendance **Pyscript** et migre l'ensemble des mécanismes associés vers des **scripts natifs Home Assistant**. 
Cette évolution réduit la surface de maintenance et élimine un custom component non essentiel.

### Suppressions

Suppression totale du composant et de tous les scripts associés :
```
custom_components/pyscript/          (module complet)
pyscript/ecs_autocorrect_offsets.py
pyscript/ecs_sauvegarde_planning.py
pyscript/ecs_restauration_planning.py
pyscript/backup_reglages_ecs.py
pyscript/restauration_reglages_ecs.py
pyscript/automatisation_id.py
```

Le bloc `pyscript:` est retiré de `configuration.yaml`.

### Migration vers scripts natifs

Chaque script pyscript est réimplémenté en YAML natif HA :
```
10_scripts/ecs/auto_correction_offsets.yaml
10_scripts/ecs/backup_and_restore/planning_sauvegarde.yaml
10_scripts/ecs/backup_and_restore/planning_restauration.yaml
10_scripts/ecs/backup_and_restore/reglages_sauvegarde.yaml
10_scripts/ecs/backup_and_restore/reglages_restauration.yaml
10_scripts/system/generation_id_automatisation.yaml
```

## 2 — ECS : horodatage du début de cycle

Introduction d'un helper dédié à la borne de départ de chaque cycle ECS :
```
07_input_datetimes/ecs/debut_cycle_timestamp.yaml
```

Une nouvelle automation assure l'alimentation de ce timestamp :
```
11_automations/ecs/log/timestamp_debut.yaml
```

L'automation `log/fin.yaml` est refactorisée en parallèle (nettoyage des commentaires, restructuration des séquences). 
La capacité des `input_text` de sauvegarde planning est portée à `max: 255` pour absorber les payloads JSON complets.

Un contrat Arsenal documente la logique d'ajustement des offsets :
```
00_documentation_arsenal/contrats/ecs/11_ajustement_des_offsets.md
```

## Synthèse

La v10.3 constitue une étape de simplification d’infrastructure et de fiabilisation du système thermique Arsenal.

Elle supprime entièrement la dépendance Pyscript au profit de scripts natifs Home Assistant, 
introduit un mécanisme autonome de préconfort thermique pour les périodes de vacances 
et renforce la robustesse du calcul des cycles ECS via un horodatage explicite du début de cycle.

Cette version introduit également une autorisation contextuelle pour l’allumage automatique du garage 
et améliore la supervision de la connectivité Vicare.
