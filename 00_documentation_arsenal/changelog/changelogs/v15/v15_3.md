## 1. NAS Search — Route d'export Markdown (webapp v1.1)

### 1.1 Nouvelle route `POST /export`

Le contrat `webapp.md` passe en v1.1 avec l'ajout d'une route d'export Markdown.

**Comportement** : `/export` accepte les mêmes paramètres que `/search` et produit une enveloppe Markdown minimale autour du stdout exact du moteur CLI. Aucune structuration sémantique (par version, par fichier, par match) n'est produite.

**Codes HTTP** : identiques à `/search` — 200 succès, 400 validation, 502 erreur moteur, 504 timeout subprocess.

**Cas de succès partiel** : si le moteur signale une troncature dans son stdout, le fichier est livré avec la troncature visible dans le bloc `text`. Aucun traitement spécial côté webapp.

**Invariants maintenus** :
- Lecture seule : aucun fichier écrit sur le NAS ; le Markdown est généré en mémoire et transmis dans la réponse HTTP.
- Autorité moteur : `/export` utilise strictement les mêmes paramètres et le même mode d'invocation que `/search`. Aucune logique grep dupliquée.
- Bornes serveur : les plafonds du moteur (résultats, contexte, durée) s'appliquent intégralement.
- Aucun shell libre : `subprocess.run(..., shell=False)`, liste d'arguments.
- Aucun état persistant : pas d'historique d'export, pas de cache, pas de stockage.

**Cohérence inter-contrats** : le contrat moteur CLI (v1.0) exclut explicitement l'export Markdown structuré parmi ses non-objectifs. L'export défini ici est non-structuré ; la cohérence est préservée. Toute évolution vers un export structuré exigerait un flag `--output-format json` côté moteur et un amendement explicite au présent contrat.

**Non-objectifs v1 enrichis** : export structuré (par version / fichier / match), export PDF / DOCX / ZIP, export HTML, historique d'exports, génération planifiée, stockage NAS des exports.

## 4. Lovelace — Palmarès pluie journalière (pluie_palmares.yaml)

Refonte de l'affichage de la carte Markdown palmarès :

| Aspect | Avant | Après |
|---|---|---|
| Style Jinja | `>` (block scalar) | `\|` (literal scalar) |
| Guard données | aucun | `or []` sur `state_attr(...)` |
| Format résultats | tableau Markdown `\| Rang \| Date \| Valeur \|` | liste linéaire avec médailles emoji (🥇🥈🥉 puis index numérique) |
| Titre | absent | `**🌧️ Palmarès pluie journalière**` intégré dans le template |
| Message vide | `Aucun record enregistré.` | `_Aucun record enregistré._` (italique) |

Le guard `or []` élimine le risque de crash Jinja si `state_attr` retourne `None` avant initialisation du sensor.

## Lecture système

v15.3 est une release de consolidation documentaire et de complétion fonctionnelle NAS. L'ajout principal est la route `/export` sur la webapp NAS Search, qui complète la boucle read-only du moteur CLI sans introduire d'état ni de logique dupliquée. Les touches Lovelace (modes, palmarès pluie) sont des ajustements d'affichage sans impact sur les contrats ou les pipelines. Aucune invariance système n'est levée.
