# Arsenal — Changelog v15.2

## Cadre

v15.2 est une version de structuration documentaire et de gouvernance du pipeline de diff NAS Arsenal.

Elle ne modifie aucun comportement métier Home Assistant.

Le périmètre concerne exclusivement :

- l’organisation du domaine `diff` ;
- la clarification des responsabilités documentaires ;
- la structuration des contrats de génération de release ;
- l’intégration patrimoniale du changelog v15.1.

---

## 1. Structuration du domaine `diff`

Le domaine documentaire NAS Arsenal `diff` est restructuré.

### Ajouts

Création des documents :

- `outils_externes/nas_arsenal/diff/diff_auto.md`
- `outils_externes/nas_arsenal/diff/diff_release.md`

Objectif :

- séparer clairement le moteur de diff automatique ;
- distinguer les releases patrimoniales des diffs techniques ;
- préparer l’industrialisation documentaire du pipeline NAS.

---

## 2. Réorganisation documentaire

Suppression de l’ancien fichier :

```text
00_documentation_arsenal/outils_externes/nas_arsenal/diff_auto.md
```

Le document est désormais relocalisé dans un domaine dédié :

```text
00_documentation_arsenal/outils_externes/nas_arsenal/diff/
```

Cette évolution matérialise officiellement le domaine `diff`
comme sous-système documentaire autonome.

---

## 3. Historisation des changelogs

Ajout du fichier :

```text
00_documentation_arsenal/changelog/changelogs/v15/v15_1.md
```

Le changelog v15.1 devient un artefact patrimonial historisé.

Le système de release Arsenal commence ainsi à produire :

- des releases ;
- des digests ;
- des changelogs versionnés ;
- des documents de gouvernance associés.

---

## 4. Lecture architecturale

v15 avait introduit :

- le pipeline MQTT d’auto-supervision ;
- le domaine `arsenal_self`.

v15.1 avait introduit :

- le pipeline métier météo pluie ;
- le moteur de release enrichi.

v15.2 consolide désormais la couche de gouvernance elle-même.

Le pipeline NAS Arsenal cesse progressivement d’être une collection
de scripts techniques et devient un véritable système documentaire
versionné.

---

## 5. Pipeline de release NAS

Le domaine `diff` est maintenant structuré autour de plusieurs couches :

| Couche | Rôle |
|---|---|
| `diff_auto.md` | génération technique des diffs |
| `diff_release.md` | doctrine release et synthèse patrimoniale |
| `digest` | vue statistique compacte |
| `full diff` | vue détaillée exploitable |
| `changelog` | lecture métier et gouvernance |

Cette séparation clarifie :

- les responsabilités ;
- les usages ;
- les publics cibles ;
- les niveaux de granularité.

---

## 6. Gouvernance patrimoniale

v15.2 confirme une doctrine désormais visible :

- les artefacts deviennent des objets gouvernés ;
- les releases deviennent historisées ;
- les diffs deviennent structurés ;
- les limitations techniques sont documentées explicitement ;
- la dette de lisibilité est pilotée plutôt que subie.

Le masquage volontaire de `.storage/core.entity_registry`
reste assumé et documenté.

---

## Impact

- Aucun changement Home Assistant runtime.
- Aucun changement Lovelace.
- Aucun changement automatisation.
- Aucun changement capteurs/helpers/scripts.
- Structuration documentaire renforcée.
- Domaine `diff` officialisé.
- Pipeline release NAS consolidé.

---

## Statut

v15.2 est une version de consolidation documentaire et de maturation du système de release Arsenal.
