# `19_button_card_templates/` — Point d'entrée

---

## Rôle de ce dossier

`19_button_card_templates/` contient les **templates UI réutilisables** (button-card)
du système : socles, génériques, et déclinaisons par dashboard. C'est du **code UI**,
pas de la doctrine.

## Où vit la doctrine

Les **règles normatives** (catalogue des socles, arbre de décision de sélection,
conventions de nommage et d'en-tête, mécanisme d'injection `!include`) ne vivent
**pas** dans ce README ni dans ce dossier, mais dans la documentation Arsenal.

Ce fichier est un simple **pointeur de découvrabilité** : il n'énumère pas les
templates et ne reproduit aucune règle. En cas de divergence, la doc centrale fait foi.
