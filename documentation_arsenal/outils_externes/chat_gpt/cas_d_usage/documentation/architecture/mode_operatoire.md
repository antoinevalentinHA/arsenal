# 🧠 ARSENAL — ChatGPT
# Mode opératoire — Architecture

## 🎯 Objet

Ce document décrit le **processus opératoire strict**
d’utilisation de ChatGPT pour la production
de documents **architecturaux Arsenal**.

Il complète le contrat sans s’y substituer.

---

## 🔁 Séquence obligatoire

### Étape 1 — Qualification explicite

L’utilisateur fournit obligatoirement :

- le **type de document** (architecture, doctrine, référence transverse),
- le **périmètre exact**,
- le **statut attendu** (normatif, figé, transverse, expérimental).

Sans cette qualification, **aucune génération n’est autorisée**.

---

### Étape 2 — Fourniture des références

L’utilisateur fournit explicitement :

- les documents Arsenal amont applicables,
- les contrats ou architectures existantes à respecter,
- les invariants à reprendre.

Toute référence manquante bloque la production.

---

### Étape 3 — Gel du périmètre

Avant rédaction, le périmètre est considéré comme :

- **clos**,
- **non extensible**,
- **non interprétable**.

ChatGPT n’ajoute aucun contenu hors périmètre.

---

### Étape 4 — Production du document

ChatGPT génère :

- un document **complet**,
- structuré,
- normatif,
- sans digression,
- sans exemple non demandé.

---

## 📤 Format de sortie

La sortie doit impérativement :

- être contenue dans **un seul bloc Markdown**,
- ne contenir **aucun texte hors bloc**,
- être **directement intégrable** dans le dépôt Arsenal.

---

## 🛑 Conditions d’échec

Toute production est invalide si :

- la séquence n’est pas respectée,
- le document dépasse son périmètre,
- un ton explicatif ou pédagogique apparaît,
- des décisions sont ajoutées.

---

## 📌 Statut

Mode opératoire opposable.
Toute déviation invalide la production.
