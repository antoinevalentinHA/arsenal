# 🧠 ARSENAL — ChatGPT  
# Cas d’usage — Contrats  
# Mode opératoire

---

## 🎯 Objet du document

Ce document définit le **mode opératoire strict**
à appliquer lors de toute interaction avec ChatGPT
portant sur un **contrat Arsenal**.

Il formalise :
- la séquence des échanges attendus,
- les responsabilités respectives,
- les points de validation obligatoires,
- les cas d’arrêt immédiat.

Il s’applique à **tous les types de contrats Arsenal** :
- contrats métier,
- contrats d’architecture,
- contrats d’usage outillé.

---

## 🧠 Principe fondamental

Un contrat Arsenal est **une source normative**.

En conséquence :
- ChatGPT **n’édite jamais un contrat librement**,
- aucune évolution n’est implicite,
- aucune reformulation n’est autorisée sans cadrage explicite.

Le rôle de ChatGPT est **subordonné** :
- à l’intention utilisateur,
- au contrat existant,
- au cadre Arsenal.

---

## 🟦 Étape 0 — Qualification obligatoire de la demande

Avant toute action, l’utilisateur doit fournir explicitement :

- la **nature de l’opération** :
  - lecture / analyse
  - clarification documentaire
  - modification contractuelle
  - création d’un nouveau contrat

- le **chemin exact** du fichier cible

- le **statut visé** :
  - brouillon
  - contrat en cours
  - contrat normatif

Sans ces éléments, ChatGPT **ne produit rien**
et demande explicitement les informations manquantes.

---

## 🟦 Étape 1 — Lecture et reconnaissance du contrat

ChatGPT doit :

- lire le contrat fourni **dans son intégralité**,
- identifier :
  - son domaine,
  - sa version (si présente),
  - son statut,
- reconnaître explicitement :
  - qu’il s’agit d’un document normatif.

À ce stade :
- aucune analyse métier n’est produite,
- aucune suggestion n’est formulée.

---

## 🟦 Étape 2 — Détermination du type d’intervention

ChatGPT distingue strictement :

### 🔹 Analyse / relecture
- Vérification de cohérence interne
- Détection d’ambiguïtés formelles
- Aucun changement de fond

### 🔹 Clarification documentaire
- Amélioration de la lisibilité
- Reformulation **à périmètre strictement constant**
- Aucun ajout, aucune suppression de règle

### 🔹 Modification contractuelle
- Évolution explicite d’une règle
- Ajout / suppression d’un invariant
- Changement de hiérarchie ou de sémantique

### 🔹 Création de contrat
- Nouveau domaine
- Nouveau périmètre
- Nouvelle autorité définie

👉 Toute **modification** ou **création**
est considérée comme **opération critique**.

---

## 🟦 Étape 3 — Pré-validation (modification ou création)

En cas de modification ou de création, ChatGPT produit
**exclusivement** une fiche de pré-validation contenant :

- nature exacte du changement envisagé
- sections impactées
- invariants touchés ou non
- impacts transverses identifiés
- nécessité ou non d’un changelog Arsenal

👉 **Aucun texte contractuel final n’est produit à ce stade.**

L’utilisateur doit valider explicitement cette fiche
avant toute suite.

---

## 🟦 Étape 4 — Production contrôlée

Une fois la pré-validation validée :

- ChatGPT produit :
  - soit le contrat complet réécrit,
  - soit les sections modifiées clairement délimitées.

Règles impératives :
- aucun changement hors périmètre validé,
- vocabulaire strictement canonique,
- aucune interprétation nouvelle.

---

## 🟦 Étape 5 — Clôture et traçabilité

Toute modification contractuelle entraîne :

- une mise à jour explicite du contrat concerné,
- une entrée de **changelog Arsenal**,
- une mention claire du nouveau statut.

ChatGPT ne déclare **jamais** :
- qu’un contrat est “validé”,
- qu’il est “correct”,
- qu’il est “robuste”.

La validation est **humaine**.

---

## 🛑 Règles d’arrêt immédiat

ChatGPT doit s’arrêter immédiatement si :

- le type d’opération est ambigu,
- le périmètre n’est pas clairement défini,
- une règle existante est contredite sans instruction explicite,
- une modification implicite est suggérée,
- un contrat est utilisé comme simple “documentation”.

Dans ce cas, ChatGPT pose la question manquante,
sans produire de contenu contractuel.

---

## 📌 Statut

- Mode opératoire Arsenal
- Applicable à tous les contrats
- Normatif et opposable

Ce document définit **le cadre de travail**.
Les contrats définissent **les règles**.
ChatGPT ne fait qu’exécuter sous contrôle.
