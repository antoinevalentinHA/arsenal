# ARSENAL — Contrat fonctionnel
## ECS — Automation `10250000000019` — Consommateur du signal canonique de fin de cycle

---

## 1. Objet

Ce contrat définit le comportement opposable de l'automation :

```
10250000000019
```

Cette automation est un consommateur du signal canonique `input_boolean.ecs_fin_cycle_signal`. Elle matérialise le pattern de consommation Arsenal : déclenchement sur signal, exécution du traitement aval, acquittement.

---

## 2. Rôle dans l'architecture

Cette automation :

- écoute le signal canonique `ecs_fin_cycle_signal`
- appelle le traitement aval configuré
- acquitte le signal après appel

Elle ne :

- ne produit pas le signal (producteur = `10250000000026`)
- ne valide pas le cycle
- ne modifie aucune donnée thermique
- ne déclenche aucune action sur la chaudière

---

## 3. Trigger

```
platform: state
entity_id: input_boolean.ecs_fin_cycle_signal
to: "on"
```

Le déclenchement est conditionné à tout passage de `input_boolean.ecs_fin_cycle_signal` à `on`. Toute autre transition est ignorée.

---

## 4. Séquence contractuelle

1. Réception du signal `ecs_fin_cycle_signal = on`
2. Appel du traitement aval (voir §5)
3. Acquittement : `input_boolean.ecs_fin_cycle_signal → off`

L'acquittement doit intervenir **après** l'appel du traitement. Il ne doit jamais précéder l'appel.

---

## 5. Traitements déclenchés

Les traitements effectivement déclenchés par cette automation sont ceux implémentés dans sa séquence. À la date du présent contrat, elle appelle :

- `script.ecs_autocorrect_offsets`

Toute extension future de cette automation doit être documentée contractuellement.

---

## 6. Acquittement

L'acquittement est réalisé par cette automation via `turn_off` sur `input_boolean.ecs_fin_cycle_signal`.

Dans l'implémentation actuelle, l'acquittement est séquentiel : il intervient après l'appel du script, sans vérification contractuelle d'un succès métier explicite retourné par celui-ci.

---

## 7. Invariants opposables

1. L'automation ne traite le signal que si `input_boolean.ecs_autocorrect_active = on`.
2. L'automation ne se déclenche que sur `ecs_fin_cycle_signal = on`.
3. L'acquittement intervient après l'appel du traitement, jamais avant.
4. L'automation n'émet pas d'action thermique.
5. L'automation ne modifie pas les données de cycle.

---

---

## 9. Notes de gouvernance

Si d'autres consommateurs du signal canonique sont introduits à l'avenir, leur coordination avec l'acquittement devra être explicitement contractualisée — l'acquittement par un consommateur éteint le signal pour tous.
