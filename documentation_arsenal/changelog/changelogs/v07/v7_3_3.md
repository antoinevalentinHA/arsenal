### 🔌 Mode panne secteur — Refactor architectural (chauffage + ECS)

Problème traité : le mode panne reposait partiellement sur des actions chauffage directes hors décision centrale, 
et la sortie de mode introduisait une latence non maîtrisée.

**Architecture mise en place :**

- **Entrée en mode panne** (`1004000000001`) : activation de `input_boolean.mode_confort_chauffage` 
comme override explicite → la décision centrale chauffage reprend immédiatement le contrôle. 
Le script `mode_panne_coupure_secteur` est recentré exclusivement sur la gestion ECS ponctuelle et les notifications. 
Exception ECS explicitement assumée et documentée pour des raisons de résilience sanitaire.

- **Sortie de mode panne** (`1004000000002`) : désactivation explicite de `input_boolean.mode_confort_chauffage` 
→ rebasculement ECS à 10 °C → notifications. La reprise chauffage est entièrement déléguée à la décision centrale.

Principe Arsenal renforcé : les événements critiques publient un état système. Les moteurs décisionnels restent souverains.

### 🔥 Décision centrale — Bypass temporisation pour override prioritaire

`input_boolean.mode_confort_chauffage` ajouté aux triggers de `10130000000090`.

Stabilisation conditionnelle introduite : override prioritaire → appel immédiat de la décision centrale (0 s) ; 
tous les autres triggers → délai de stabilisation conservé (2 s).

Signal d'intention critique non soumis à la temporisation générique d'absorption de bruit.

### 🧠 Chauffage — Verrouillage sémantique standby / repos nominal

Deux corrections structurelles sur la couche décision + UI :

**Standby vs blocage absolu** : `standby_force` était évalué après `chauffage_non_autorise` dans `sensor.chauffage_raison_calculee` 
→ en hystérésis, le standby était écrasé et interprété comme blocage absolu côté UI. Correction : `standby_force` évalué en priorité. 
Retrait de `standby_force` de la liste des blocages absolus dans `carte_chauffage_synthese`.

**Neutre ≠ attente thermique** : `Neutre + programme Confort` = repos nominal (confort satisfait, brûleur inactif). 
Correction UI : `Neutre + Confort` → "Confort satisfait / repos" ; `Neutre + ≠ Confort` → "Attente confort".

Mapping UI final dans `carte_chauffage_synthese` :
- 🔥 Brûleur actif → **Actif** (rouge)
- ⛔ Blocage absolu (poêle, aération, fenêtre, vacances, système) → **Bloqué** (orange)
- 🔒 `standby_force` → **Attente • thermique** (bleu)
- 🟢 Programme Confort, brûleur inactif → **Confort** (vert)
- ⏳ Autres cas absence/présence → **Attente • confort / protection**

`carte_chauffage_intention` : libellés Eco enrichis par la raison métier (`Sobriété • protection thermique`, `Sobriété • absence`).
