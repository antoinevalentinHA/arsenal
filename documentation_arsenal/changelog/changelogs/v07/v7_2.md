## 🔥 Chauffage — Intention **Neutre** & fin des décisions implicites de présence

- Introduction explicite du mode **Neutre** : autorisation thermique **sans action**
  (pas de chauffe “par principe”).
- Décision centrale : ajout d’un garde-fou “**Stop si mode neutre**”
  (neutralité = abstention active, pas un “pseudo-confort”).
- Décision “mode” : re-priorisation du blocage **Poêle** (contrainte dure)
  avant l’interprétation de la présence.
- Présence : suppression de toute projection implicite en **Confort** ;
  la présence devient compatible avec **Neutre**
  (autorisé mais non déclencheur).
- Traduction UI du contrat Chauffage V3 :
  - **attente confort** = “Neutre / autorisé sans action”
  - **attente protection** = absence / protection bâti
  - **activité réelle** = état effectif

### ⚠️ Points de vigilance (Chauffage)
- **Perception UI** : tout écran/diagnostic qui supposait “présence = confort”
  doit désormais accepter “présence = neutre”.
- **Décision muette** : vérifier que “Neutre” n’est jamais interprété comme une panne
  (c’est une abstention volontaire).
