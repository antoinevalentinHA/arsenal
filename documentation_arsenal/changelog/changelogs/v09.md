## 🖥️ 4) Chauffage — Robustesse UI (notifications persistantes)

### Problème corrigé
Notifications persistantes Chauffage pouvant rester obsolètes si un script en `mode: restart`
est interrompu avant disqualification.

### Correctif appliqué
- Suppression complète de toute logique UI `persistent_notification.*` du script `chauffage_decision_centrale`
- Introduction d’une automation UI dédiée, idempotente :
  - **ID :** `10240000000019`
  - **Source :** `input_select.chauffage_dernier_mode_decide`
  - **Stratégie :** dismiss universel → create contrôlé (reconstruction fiable)

> Principe renforcé :
> la notification persistante est une projection UI, découplée de la séquence d’exécution d’un script.
