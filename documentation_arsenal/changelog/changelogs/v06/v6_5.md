## 🧠 1. CHAUFFAGE — ALIGNEMENT DÉCISIONNEL (V3 PRO)

* contextes bloquants (aération, fenêtres, poêle, vacances) → **Eco imposé**
* la présence reléguée au rang de confort opportuniste
* la présence ne peut jamais forcer un passage en Confort

Alignement strict entre :

* `chauffage_decision_centrale`
* `sensor.chauffage_mode_calcule`
* `sensor.chauffage_raison_calculee`

Corrections explicatives :

* refonte complète du système de justification
* la raison reflète désormais la **cause dominante réelle**
* suppression des raisons trompeuses de type `presence_on`

Principe cardinal verrouillé :

> La présence n’est **JAMAIS** une autorité thermique.

Impact :

* comportement attendu rétabli sans exception
* cohérence totale décision / explication / UI
* réduction forte du risque de régression

## 🌿 3. ÉCLAIRAGE JARDIN — CONSOLIDATION MATIN / SOIR

Principes verrouillés :

* séparation stricte décision / contexte / action
* capteurs déterministes et stateless
* automatisations événementielles et idempotentes
* aucun couplage direct MATIN / SOIR

### 🌅 MATIN — Fenêtre lumineuse

* décision basée sur :

  • autorisation utilisateur
  • temps civil
  • bornes journalières
  • état solaire

* présence : conditionne l’activation, jamais la décision

* extinction immédiate à la sortie de fenêtre

### 🌆 SOIR — Cycle temporel

* existence du cycle basée sur :

  • coucher du soleil (+ offset)
  • bornes temporelles
  • autorisation utilisateur

* présence : déclenche l’allumage, n’autorise pas le cycle

* extinction pilotée par autorisation métier de confort

* extinction irréversible pour le cycle
