# 🧠 Arsenal — Positionnement système

## Objet

Ce document fixe le positionnement architectural réel du système Arsenal.

Il définit :
- la catégorie technique exacte du système,
- ce qu’Arsenal n’est plus,
- ce qu’Arsenal est devenu,
- son périmètre fonctionnel réel.

---

## Ce qu’Arsenal n’est PAS

Arsenal n’est plus :
- une installation domotique classique
- une simple configuration Home Assistant
- une GTB résidentielle simplifiée
- un système événementiel

---

## Ce qu’Arsenal est

Arsenal est un :

**Système Cyber-Physique Résidentiel de Supervision Multi-Domaines
à base de Modèle Logiciel Dominant**

Caractéristiques fondamentales :
- modèle logiciel majoritaire
- état interne riche et persistant
- séparation décision / action / perception
- multi-domaines natif
- gouvernance documentaire

---

## Classification technique

Catégorie :
- Residential Cyber-Physical Supervisory System (R-CPSS)

Sous-catégorie :
- Model-Driven Supervisory System

---

## Finalité

Arsenal vise :
- la stabilité thermique et énergétique
- l’observabilité permanente
- la souveraineté décisionnelle
- la transposabilité multi-logements
